Standard Response Structure

Use this format across all APIs:

Login API

  • OTP Sent Successfully (Registered User)

  • User Not Registered

OTP Verify API

  • OTP Verified Successfully (Proceed to Login)

    access_token will have following data in encrypt format
    access_token Decoded

  • Invalid OTP

  • OTP Expired

User Registration API

  • User Registered Successfully

    Note: Use HTTP 201 Created instead of 200 to indicate a new resource was created.

    The has_updated_password field is set to false, and the is_first_login field is set to true while registering a new user.

  • User Already Registered

    Note: success: false is more semantically correct because registration did not occur.

    Keep status: 200 for “successful request” (API worked), but the action was not performed.

  • Validation Errors
