Landing / Home Page
  • When the user first visits the Shidori web app, they are directed to the Home Page.

  • The system asks the user for their location:

    • Automatic detection via browser/location services.
    • Manual entry is required if automatic detection fails or the user prefers to enter a location manually.
  • Based on the location, the app can serve personalised recommendations and available menu items from nearby kitchens.
Browsing & Adding Items to Cart
  • Users can browse menu items without logging in.
  • When a user tries to add an item to the cart or proceed to checkout, the system checks if the user is logged in.

  • If the user is not logged in:

    They are prompted to log in or register to continue with the order.

Registration / Sign-Up

  • The registration form collects the user’s basic details, including:

    • Name
    • Email Address
    • Mobile Number
    • Date of Birth

  • After entering the required details and clicking the Sign-Up button:

    • The user receives a registration confirmation.
    • The user is redirected to the Login page.

  • If the user is already registered:

    • A message will be displayed: “Already registered. Please log in using your email or mobile number.”
    • The user will be redirected to the Login page.
Login & OTP Verification
  • The user enters their registered email/mobile number and clicks Send OTP.
  • The Verify OTP screen appears.
  • An OTP is sent to the user’s registered email/mobile.
  • The user enters the OTP and clicks Submit.

  • The system validates the OTP:

    • If the OTP is incorrect, an error message is displayed.
    • If the OTP is correct, proceed to the next step.
First-Time Login & Onboarding

  • After OTP verification, the system checks if it is the user’s first login.

    • If yes, redirect to the onboarding screens introducing Shidori.
    • Users navigate through the introduction screens using the Next button.
    • The last screen has a Start button.
  • Clicking the Start button redirects the user to their Dashboard/Home page, now ready to place an order.

Add to Cart Page

The Add to Cart (Shopping Bag) page is the main review and confirmation step before checkout. It lets users see all items they’ve selected, adjust quantities, apply coupons, choose the delivery address, optionally add recommended items or a donation, and then proceed to payment in a single, focused flow.

Page purpose and layout

  • The page is divided into three vertical zones:

    1. Cart Items Area on the left (all dishes in the cart).
    2. Order Summary Sidebar on the right (coupon, address, totals, checkout).
    3. Donation & Policy Section at the bottom (optional donation and detailed cancellation policy).​
  • The top of the page shows the Shopping Bag title, the item count, and a Clear Cart button to reset the cart in one action.​

Cart items workflow

  • Each cart line shows the dish image, name, description, star rating, unit price, quantity control, and total, so users can understand and manage every item easily.​
  • The initial quantity comes from wherever the user last added/updated that item (menu page or detail page).

  • Users can:

    • Increase quantity with the plus button, which updates the line total and overall cart total in real time.
    • Decrease quantity with the minus button; if the quantity reaches zero, the item is removed from the cart.
  • Any change in quantity immediately updates the cart item count and the Cart Total / Total figures in the sidebar, keeping pricing transparent.​

Recommendations / Add more items

  • Below the main list cart items, a strip shows recommended dishes (e.g., Butter Naan, another curry), each with an image, name, short description, rating, price, and an “+ Add” button.
  • Clicking “+ Add” adds that item to the cart with a default quantity of 1, updates totals, and visually confirms the addition (for example, button state change or toast), allowing quick upsell without leaving the cart.​

Coupon, address and totals workflow

  • In the right sidebar, the Coupon Code panel lets users enter a code and tap Apply.

    • On valid codes, the Discount value and Total are recalculated and updated; an error state appears for invalid/expired codes.​

  • The Deliver block shows the currently selected address (name, address, phone) and a navigation arrow.

    • Tapping it opens the address management flow, where users can change or add addresses; when they return, the selected address is reflected here.​

  • The Cart Total panel displays a breakdown:

    • Cart total (sum of item line totals).
    • Delivery charge.
    • Discount (from coupons/offers).
    • Total (final payable amount).
  • The Checkout button is the primary call‑to‑action, moving the user from this review page to the checkout/payment flow once they are satisfied with items, address, and totals.​

This information helps users feel secure and informed about their purchase, making it easier for them to complete the order without hesitation.

The pop-up asks two questions: “How would you rate your experience at Shidori?” and “How would you rate our Delivery Partner Shidori?”, each paired with a 5‑star rating control so users can separately evaluate the overall Shidori experience and the delivery partner’s performance.

Under the star ratings, a “Write your comment” text box lets users leave optional written feedback about food quality, packaging, punctuality, or delivery behaviour. At the bottom, a prominent Submit button sends the selected star ratings and comment, and a closing line explains that “Your ratings will help other privileged members in discovering better places,” reinforcing that their feedback benefits the whole community.

User Profile Page

Profile Page – overall description

  • The Profile page is the central place where users can view and manage their personal account information, preferences, and app settings.
  • At the top, a Wallet Money button gives quick access to the user’s in‑app wallet so they can view their current balance or add money for future orders.
  • A notification icon opens the notifications view, where users can see recent alerts related to orders, offers, and account updates.
  • On the right side of the header, the profile thumbnail shows the user’s avatar; tapping it allows them to change their profile photo.​
  • Just below the header, a Back link lets users return to the previous screen, typically the home or main menu.​

  • The main body of the page is split into two columns. On the left, a vertical navigation menu lists all profile-related sections:​

    • Account Settings
    • Order History
    • Addresses
    • Help Center
    • Promocodes & Offerings
    • Subscription
    • Invite Friends
    • Privacy Policy
    • Logout
    • Delete Account
  • On the right, a content area displays the details and controls for whichever menu item is currently selected.
  • When Account Settings is active, this panel shows the user’s basic profile information and notification preferences.​

  1. Account Settings

    • When Account Settings is selected in the left menu, the right panel becomes the user’s personal profile editor.​
    • At the top of this panel, the user’s profile picture, full name, and email address are shown in a compact header so core identity information is always visible.​
    • The profile picture can be updated by tapping or clicking on it and selecting a new image from the device.​
    • Below this header, there are editable fields for Name, Email, Birthday, and Phone No. so users can keep their core account information up to date.​
    • Any changes saved here are used across orders, communications, and receipts.​
    • Inputs such as Birthday and Phone No. should use clear, specific formats (for example, a date picker and formatted phone field) to reduce errors.​
    • Further down, a Notifications block lets users manage how they receive updates from the app.​ A short helper line, such as “View and change your notification settings”, explains the purpose of this section.​

  2. Order History

    • The Order History section is accessed from the left-side Profile menu and shows a list or grid of all orders that have already been delivered to the user.​
    • Each entry focuses on completed orders, so users can quickly review what they have previously ordered and how much they paid.​
    Order cards in Order History
    • Every order is represented as a card containing key details such as Order ID, delivery date and time, main items, total quantity, and total price.​
    • Visual hierarchy (order number at the top, items and prices in the middle, total at the bottom) helps users scan multiple cards easily.​
    • Only orders with a Delivered or Completed status appear here, ensuring the list is clearly about past, fulfilled orders.​
    Navigation to Item Delivered page
    • When the user clicks or taps any order card in Order History, the app opens the Item Delivered page for that specific order.​
    • The system passes the selected order’s ID to the Item Delivered view so that all details shown there (items, prices, and status) match the chosen card.​
    Item Delivered page and rating entry
    • The Item Delivered page shows detailed information for that order: ordered items with images, quantities, individual prices, total amount, and a clear “Delivered” status.​
    • If the user has not yet rated this order, the page displays a prominent call‑to‑action (for example, “Rate this Order” or a star‑rating placeholder) that leads to the Rating page.​
    • Selecting this action redirects the user to the Rating page, where they can add a star rating, optionally confirm their name and meal name, and write a text review before submitting.​
    Showing existing rating on Item Delivered page
    • If the user has already submitted a rating and review for the order, the Item Delivered page shows the existing star rating and review text directly under the order details.​
    • This makes it clear that feedback has been recorded and lets the user quickly recall what they thought of that particular meal.​
    Updating an existing rating
    • Alongside the displayed rating, an action such as “Edit Rating” or an editable star control is shown so that the user can update their feedback if their opinion changes.​
    • Clicking this action takes the user back to the Rating page, with their previous rating and review pre‑filled in the form, so they only need to adjust the parts they want to change.​
    • After the user resubmits the updated rating, the revised stars and review text are saved and immediately reflected again on the Item Delivered page and, where relevant, in the Customer Reviews area.
